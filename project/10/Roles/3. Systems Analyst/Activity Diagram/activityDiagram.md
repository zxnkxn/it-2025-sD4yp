# Описание сценариев использования — cценарий игрока

## Цель
Просмотр персонализированных рекомендаций, выбор игры, выставление оценки и написание ревью.

## Действующие лица
- **Player (Игрок)** — основной пользователь, взаимодействует с системой для получения рекомендаций и управления игровым опытом.

## Основной поток событий
1. Игрок авторизуется в системе.
2. Система проверяет учетную запись.
3. При успешной авторизации игрок попадает на вкладку **Explore**.
4. Система загружает рекомендации с учетом:
   - предпочтений игрока (Favorite Genres);
   - истории оценок (Ratings);
   - ранее отмеченных любимых игр.
5. Игрок выбирает интересующую игру.
6. Открывается новая страница игры, где отображается:
   - описание и обложка игры;
   - список отзывов и оценок других пользователей.
7. Игрок выставляет оценку игре (1–5).
8. Система сохраняет оценку и обновляет отображение данных.
9. Игрок может написать комментарий (ревью):
   - если игрок хочет написать ревью — система сохраняет комментарий;
   - если игрок не хочет писать ревью — шаг пропускается.
10. При изменении предпочтений или скролле в списке рекомендаций система обновляет рекомендации динамически.

## Альтернативные потоки
- Если игрок не найден при авторизации — выводится сообщение об ошибке, и дальнейшие действия невозможны.
- Если игрок решает не писать ревью — процесс продолжает работу без сохранения комментария.

## Потоки данных
- Входные данные: email и пароль для авторизации, выбранные жанры и оценки, текст ревью.
- Выходные данные: список рекомендованных игр, информация об игре, обновленные оценки и отзывы.
