# Диаграмма вариантов использования

## Действующие лица

1. **Player (Игрок)** — просматривает рекомендации, выставляет оценки, оставляет отзывы, управляет предпочтениями и профилем.
2. **Developer (Разработчик)** — публикует свои игры на платформе.
3. **Moderator (Модератор)** — проверяет публикации, управляет модерацией и мониторингом системы.

---

## Варианты использования для Player

- **Просмотр рекомендаций (Explore)** — игрок видит список рекомендуемых игр и может выбрать интересующую.  
- **Управление предпочтениями (Preferences)** — игрок выбирает любимые жанры, которые влияют на рекомендации.  
- **Просмотр и редактирование профиля (Profile)** — игрок просматривает статистику, изменяет пароль, удаляет аккаунт.  
- **Просмотр информации об игре (ViewGame)** — открытие нового окна с деталями игры, включая описание, обложку и отзывы других пользователей.  
- **Выставление оценки игре (RateGame)** — выставление числовой оценки от 1 до 5.  
- **Написание ревью (WriteReview)** — игрок оставляет комментарий к оценке игры (только один комментарий на игру).  

---

## Варианты использования для Developer

- **Публикация игры (PublishGame)** — разработчик отправляет информацию об игре на платформу для публикации.

---

## Варианты использования для Moderator

- **Модерация игр (ModerateGame)** — проверка игр, предоставленных разработчиками, принятие решения о публикации.  
- **Мониторинг состояния системы (SystemMonitor)** — контроль работоспособности и активности пользователей, статистики системы.

---

## Взаимосвязи между вариантами использования

- **Include**: `WriteReview` включает `RateGame`, так как невозможно оставить отзыв без оценки.  

---

## Основные бизнес-потоки

1. Игрок авторизуется → открывает вкладку **Explore** → просматривает рекомендации → выбирает игру → открывает детали → выставляет оценку и (опционально) пишет ревью.  
2. Игрок изменяет предпочтения → вкладка **Explore** обновляется с учётом новых жанров.  
3. Разработчик создаёт игру → отправляет на публикацию → модератор проверяет → игра становится доступной игрокам.  
4. Модератор наблюдает за состоянием системы и проверяет активность игроков/разработчиков.
