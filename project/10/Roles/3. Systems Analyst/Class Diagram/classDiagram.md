# Модель предметной области

Модель предметной области описывает ключевые сущности системы рекомендаций в игровой индустрии, а также взаимосвязи между ними. Модель отражает основные бизнес-объекты, используемые в системе, и служит основой для формирования функциональных и нефункциональных требований.

# Пользователи системы

Базовой сущностью является User, представляющая зарегистрированного пользователя платформы. Пользователь характеризуется уникальным идентификатором, никнеймом, электронной почтой, датой регистрации и аватаром.

В системе выделяются три специализированные роли пользователей:

1. Player — игрок, получающий рекомендации, оценивающий игры и оставляющий отзывы;
2. Developer — разработчик, публикующий игры на платформе;
3. Moderator — модератор, осуществляющий проверку и контроль публикуемого контента.

Связь между сущностями реализована с использованием наследования: каждая роль является специализированным видом пользователя.

# Игры и жанры

Сущность Game описывает игру, представленную на платформе. Игра содержит идентификатор, название, описание, дату релиза и ссылку на обложку.

Сущность Genre используется для классификации игр по жанрам. Одна игра может относиться к нескольким жанрам, и один жанр может быть связан с несколькими играми, что отражается связью многие-ко-многим.

Для хранения настроек игрока используется сущность UserPreferences. Она агрегирует предпочтения пользователя и принадлежит конкретному игроку.

Игрок может выбрать несколько любимых жанров, которые используются системой рекомендаций для персонализации контента. Связь между предпочтениями и жанрами реализована как многие-ко-многим.

Сущность Rating представляет числовую оценку, выставляемую игроком игре. Каждая оценка связана с одним игроком и одной игрой. Один игрок может выставить несколько оценок разным играм, а одна игра может иметь множество оценок.

Сущность Review описывает текстовый отзыв пользователя. Отзыв также связан с конкретным игроком и конкретной игрой. Игрок может написать только одно ревью для каждой игры, но может оставлять ревью для разных игр.

Связь между оценкой и отзывом является агрегацией: отзыв может существовать только в контексте оценки, при этом наличие отзыва не является обязательным.

# Публикация и модерация контента

Разработчик может публиковать несколько игр, при этом каждая игра связана с одним разработчиком. Модератор осуществляет проверку и контроль игр, принимая решение об их публикации, что отражено ассоциацией между модератором и играми.

# Вывод

Представленная модель предметной области охватывает основные сущности системы рекомендаций, их атрибуты и взаимосвязи. Модель используется как концептуальная основа для проектирования вариантов использования, сценариев взаимодействия и архитектурных решений системы.
